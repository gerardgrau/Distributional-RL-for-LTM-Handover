# results/

All non-source artifacts produced by training, evaluation, and analysis.
Every subdirectory has its own `.gitignore` that excludes the bulky outputs
by default, so the layout below describes "what lives here" rather than
"what is tracked".

## Layout

```
results/
├── benchmarks/                       # Per-run training output (main.py target)
│   └── bmk_<YYYY-MM-DD>_<num>_<desc>/
│       ├── train/                    # Per-episode CSV log per (agent, seed)
│       ├── eval/                     # Post-training frozen evaluation
│       ├── models/                   # <agent>_seed<N>.pth + <agent>_best.pth
│       ├── figures/                  # learning_curves, metrics_grid, etc.
│       ├── animations/               # Per-UE dashboard MP4s
│       ├── config.yaml               # Copy of the active config
│       └── metadata.json             # Wall time, device, full config
├── final_metrics/                    # Cross-run master comparison set
│   ├── *_summary.csv                 # Per-agent final-metric summaries
│   ├── paper_*.csv                   # Reference numbers from the paper
│   ├── plots/                        # Every generated comparison figure
│   │                                 #   (master_bar_plots*, master_radial_plot,
│   │                                 #    risk_frontier*, sweep/quantile/atari PNGs)
│   ├── sweeps/                       # Archived sweep / study CSVs
│   └── reform_a/                     # Self-contained reformation study
├── animations/                       # Ad-hoc one-off animations
├── analysis/                         # Ad-hoc plots / scratch analysis
├── models/                           # Loose model checkpoints (legacy path,
│                                       new training writes inside bmk/)
└── simulations/                      # Raw pickled simulator state (huge)
```

## Conventions

- **Benchmark folders** follow `bmk_YYYY-MM-DD_<num>_<description>`.
  `main.py` auto-increments `<num>` to avoid collisions on the same date.
- **Master comparison plots** (`final_metrics/plots/`) are regenerated by
  `src/tools/generate_final_plots.py` from whatever `*_summary.csv` files are
  present at the `final_metrics/` top level; the `plot_*.py` figure scripts also
  write here, and the sweep-CSV writers write into `final_metrics/sweeps/`.
- Older one-off animations or analysis pngs are safe to `trash` once the
  per-benchmark `animations/` subfolder has the equivalent — that path is
  the canonical home for new ones.
